# TODO NewDiary

Статус на 23.04.2026: проект находится на стадии рабочего веб-прототипа для сбора обратной связи после событий программы мероприятия, личной рефлексии участников и аналитики для организаторов и кураторов.

## Что уже сделано

- Frontend на React + Vite с маршрутизацией через React Router.
- Ролевой доступ для `participant`, `curator`, `organizer`, `admin`: отдельные маршруты, guards и базовая RBAC-логика.
- Регистрация участников на публичные заезды через `/register` и список открытых событий из API.
- Backend на Express с JSON API, cookie auth, dev auth, magic links и setup-страницей первого администратора.
- PostgreSQL-схема для пользователей, заездов, групп, программ, дней, событий, дневниковых записей, дневных рефлексий, опросов, публикаций, аналитики, AI-отчетов и audit log.
- Кабинет администратора для управления пользователями, заездами, назначениями, регистрацией и базовой безопасностью.
- Кабинет организатора:
  - список и создание заездов;
  - редактирование параметров заезда и регистрации;
  - рабочее пространство программы;
  - дни программы;
  - события программы;
  - параллельные потоки;
  - единый табличный конструктор расписания;
  - создание, редактирование, удаление и активация событий;
  - обзор групп и участников.
- Участник:
  - чистый кабинет участника без технических бейджей и лишнего dashboard-шума;
  - верхняя панель участника с контекстом заезда, навигацией `Состояние` / `Узнать себя` / `Динамика` и меню аккаунта с выходом;
  - основной раздел `Состояние` для дневника после событий;
  - mobile-first экран дневника для быстрого ввода с телефона;
  - отдельный prop-driven компонент `StateScalePicker` для выбора состояния после события;
  - нейтральная шкала состояния с 7 состояниями и 3 зонами: выгорание, интеграция, дистресс;
  - `arc`-вариант шкалы как спидометр со слайдером, нейтральным предпросмотром до выбора и разведенными подписями зон;
  - последовательное заполнение событий дня через раскрываемые карточки без автоскролла и компактные строки остальных событий;
  - проблемная motion-механика карточек отключена: переключение `Назад` / `Далее` остается предсказуемым и не промахивается по экрану;
  - комментарии и отметка сложности оценки;
  - итоговая рефлексия открывается отдельным ритуальным действием после заполнения событий;
  - раздел `Узнать себя` как подготовленная входная карточка под психологический тест статуса идентичности;
  - экран `Динамика`;
  - подробная аналитика дня в раскрываемом блоке, чтобы не перегружать основной поток;
  - состояния `unpublished` и `published-empty`, если программа еще не опубликована или опубликована без событий.
- Куратор:
  - routed-раздел пересобран вокруг метафоры `Пульс дня` на реальных PostgreSQL/API-агрегатах;
  - API-контракт `GET /api/curator/sessions/:sessionId/groups/:groupId/dashboard` расширен полями `dataState`, `eventPulse`, `participantRows`, `reflectionPrep`, `organizerBrief`;
  - UI показывает `Пульс дня`, `Партитуру группы`, бриф к дневной рефлексии и сигналы для живого разговора с организаторами;
  - пропуски в траектории остаются `null`, без замены на нейтральное значение `3`;
  - данные `risk_signals`, `comment_clusters`, `ai_reports` строго scoped по группе куратора;
  - Storybook-покрытие добавлено для `Curator/Pulse Dashboard`: `Normal`, `Mobile`, `NoResponses`, `NoPublishedProgram`, `HighRisk`, `EmptyClustersAndAi`.
- Аналитический слой:
  - rule-based метрики по состояниям и прогрессу заполнения;
  - групповые, кураторские и организаторские сводки;
  - заготовки под AI-отчеты и дальнейшую кластеризацию комментариев.
- Storybook подключен для компонентов и страниц, есть controls, viewport presets и `@storybook/addon-a11y`.
- Есть базовая подготовка к production: env-настройки, PostgreSQL scripts, блокировка demo seed в production, `prod:init`.

## Приемка для публикации программы

- Draft-программа не видна участнику: дневник показывает состояние `unpublished`.
- После публикации участник видит дни и события опубликованной программы.
- Опубликованная программа без событий показывает состояние `published-empty`.
- Организатор видит понятный статус программы и не может случайно опубликовать уже опубликованную программу повторно.
- Storybook покрывает кнопку публикации и состояния `draft`, `published`, `saving`, `error`.
- `npm run build` проходит без ошибок.
- `npm run build-storybook` проходит без ошибок.

## Приемка для mobile-first участника

- Дневник открывает первое событие без выбранного состояния, а если все события заполнены - первое событие дня.
- Открытую карточку можно свернуть, а клик по другой карточке раскрывает ее и закрывает предыдущую.
- Свернутые события отображаются плотными компактными строками со временем, названием, статусом заполнения, `aria-expanded` и видимым focus.
- Открытая карточка показывает крупный выбор состояния, а комментарий и отметку "сложно оценить" - только после выбора состояния.
- Кнопки "Назад" и "Далее" переключают события без автоперехода после выбора состояния, автопрокрутки и анимационного промаха.
- Нейтральный текст шкалы короткий: без длинного объяснения модели до выбора.
- Рефлексия заблокирована до заполнения всех событий, затем открывается по кнопке `Открыть итог дня`; уже заполненная рефлексия открывается сразу.
- Графики, распределение и автосводка лежат в раскрываемом блоке.
- Storybook содержит мобильные сценарии `MobileQuickInput`, `SequentialIncompleteDay`, `CompletedMobileDay`, `ReflectionLocked` и состояния `UnpublishedProgram` / `PublishedEmptyProgram`.
- `Participant/Diary` проверяется на Storybook viewport `Mobile 390` и `Desktop 1280`.
- На `Mobile 390` и `Desktop 1280` свернутые карточки идут компактным списком без чрезмерных отступов.
- `npm run build` проходит без ошибок.
- `npm run build-storybook` проходит без ошибок.

## Приемка для компонента выбора состояния

- `StateScalePicker` показывает состояния в порядке выгорание -> интеграция -> дистресс.
- Желтый вариант `engaged` остается в зоне интеграции, оранжевый `overstimulated` и красный `panic` - в зоне дистресса.
- Вариант отображения выбирается через prop `variant`: `arc`, `zones`, `compact`; `arc` работает как спидометр со слайдером.
- В `arc` до первого выбора показывается нейтральный preview в центре шкалы без сохранения ответа.
- В `arc` подписи зон `выгорание`, `интеграция`, `дистресс` не пересекаются с дугой на `Mobile 390` и `Desktop 1280`.
- В `arc` выбор вызывает `onChange(stateId)` после отпускания слайдера или завершения клавиатурного выбора и не меняет backend/API-контракт дневника.
- `arc` доступен как native slider с `aria-valuetext`; `zones` и `compact` остаются keyboard radiogroup с `aria-checked`.
- Storybook `Participant/StateScalePicker` содержит Controls для `variant`, `value`, `animated`, `disabled`, `showDescriptions` и story `ArcNeutralPreview`.
- Проверить Storybook на viewport `Mobile 390` и `Desktop 1280`.
