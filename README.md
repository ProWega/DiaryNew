# NewDiary

Веб-прототип платформы дневников состояний участников форума с ролевым доступом, аналитикой по группам и mock-слоем для ИИ-отчётов.

## Что это

Проект реализует frontend-прототип для сценариев:

- участник отмечает состояние по событиям дня и заполняет итоговую рефлексию;
- куратор смотрит динамику своей группы, сигналы риска и тематические кластеры;
- организатор анализирует заезд целиком, проблемные события и типологии;
- администратор управляет контуром доступа, безопасностью и аудитом.

Сейчас приложение работает на `React + Vite`, использует маршрутизацию, mock JSON/API-слой и RBAC-проверки по роли, заезду и группе.

## Стек

- React 19
- Vite
- React Router
- Mock JSON/API поверх `localStorage`
- SVG-графики без сторонних chart-библиотек

## Возможности

- маршруты по ролям и сценариям вместо локального переключателя экранов;
- ролевой доступ для `participant`, `curator`, `organizer`, `admin`;
- ограничение доступа по `session` и `group`;
- дневник участника с редактированием состояний и рефлексии;
- dashboard куратора с heatmap, рисками и темами комментариев;
- dashboard организатора с группами, типологиями и ИИ-отчётами;
- admin-раздел с RBAC-матрицей и аудитом;
- mock API, который имитирует backend и сохраняет данные в браузере.

## Быстрый старт

```bash
npm install
npm run dev
```

Dev-сервер поднимается на всех интерфейсах:

```bash
vite --host 0.0.0.0
```

Production build:

```bash
npm run build
```

## Демо-пользователи

Верхняя панель позволяет переключаться между mock-пользователями:

- участник;
- куратор;
- организатор;
- администратор.

Так можно быстро проверить маршруты и RBAC без реальной авторизации.

## Основные маршруты

- `/participant/session/:sessionId/today`
- `/participant/session/:sessionId/dynamics`
- `/curator/session/:sessionId/group/:groupId`
- `/organizer/session/:sessionId`
- `/admin/security`

## Архитектура

### UI

- `src/main.jsx` — точка входа
- `src/AppRouter.jsx` — корневой роутер
- `src/components/AppLayout.jsx` — общий layout приложения
- `src/pages/` — routed pages
- `src/views/` — презентационные экраны

### Данные и доступ

- `src/api/mockServer.js` — mock backend
- `src/api/jsonApi.js` — клиентский API-слой
- `src/api/hooks.js` — hooks для загрузки данных
- `src/auth/AuthContext.jsx` — текущий пользователь и bootstrap
- `src/rbac/permissions.js` — правила доступа

### Аналитика

- `src/lib/metrics.js` — расчёт метрик и rule-based summaries
- `src/components/Charts.jsx` — SVG-графики и heatmap

## Как сейчас устроены ИИ-отчёты

На текущем этапе ИИ-слой частично mock-овый:

- отчёты организатора и сводки куратора приходят как заранее заданные данные;
- дневной "портрет" участника считается rule-based на клиенте из событий и состояний;
- внешнего LLM-вызова пока нет;
- embeddings, кластеризация комментариев и фоновая генерация ещё не подключены.

То есть это хорошая основа для следующего шага, но ещё не production-ready AI pipeline.

## Хранение данных

Сейчас данные сохраняются в `localStorage`, чтобы было удобно тестировать прототип без backend.

Это означает:

- изменения дневника сохраняются между перезагрузками страницы;
- данные локальны для текущего браузера;
- серверной синхронизации пока нет.

## Что можно развивать дальше

- подключить реальный backend и PostgreSQL;
- добавить настоящую аутентификацию;
- вынести AI Analytics в отдельный сервис;
- реализовать импорт CSV/XLSX;
- добавить реальные уведомления и сигналы риска;
- подключить версионирование ИИ-отчётов.

## Статус

Проект находится в стадии интерактивного sandbox-прототипа.

Он уже подходит для:

- демонстрации UX и логики ролей;
- обсуждения структуры данных;
- подготовки backend/API-контракта;
- следующего шага по интеграции реального ИИ-контура.
